# 환경 변수 설정 가이드

이 문서는 프로젝트에 필요한 환경 변수를 설명합니다.

## .env.example 파일 생성

프로젝트 루트에 `.env.example` 파일을 생성하고 다음 내용을 추가하세요:

```env
# 환경 변수 예제 파일
# 이 파일을 복사하여 .env 파일을 생성하고 실제 값을 입력하세요.
# .env 파일은 .gitignore에 포함되어 버전 관리에서 제외됩니다.

# ============================================
# 백엔드 API 포트 설정
# ============================================
# Auth Service 포트 (기본값: 8081)
VITE_AUTH_API_PORT=8081

# User Service 포트 (기본값: 8082)
VITE_USER_API_PORT=8082

# Audit Service 포트 (기본값: 8083)
VITE_AUDIT_API_PORT=8083
```

## 사용 방법

1. `.env.example` 파일을 복사하여 `.env` 파일 생성:
   ```bash
   cp .env.example .env
   ```

2. `.env` 파일을 열어서 필요한 경우 값을 수정합니다.

3. 개발 서버를 재시작합니다:
   ```bash
   npm run dev
   ```

## 주의사항

1. **VITE_ 접두사 필수**: 모든 환경 변수는 `VITE_` 접두사로 시작해야 합니다.
   - Vite는 `VITE_` 접두사가 있는 환경 변수만 클라이언트에 노출합니다.
   - 보안상 중요한 정보는 환경 변수에 저장하지 마세요.

2. **개발 서버 재시작 필요**: 환경 변수를 변경한 후에는 개발 서버를 재시작해야 합니다.

3. **빌드 타임 반영**: 운영 환경에서는 빌드 타임에 환경 변수가 포함됩니다.
   - 빌드 후 환경 변수 변경은 반영되지 않습니다.

4. **보안**: 
   - 민감한 정보(API 키, 비밀번호 등)는 환경 변수에 저장하지 마세요.
   - 클라이언트 측 코드에 노출되므로 보안에 취약합니다.
   - `.env` 파일은 `.gitignore`에 포함되어 있어 버전 관리에서 제외됩니다.

## 환경 변수 설명

| 변수명 | 설명 | 기본값 | 필수 |
|--------|------|--------|------|
| `VITE_AUTH_API_PORT` | Auth Service 포트 번호 | 8081 | 아니오 |
| `VITE_USER_API_PORT` | User Service 포트 번호 | 8082 | 아니오 |
| `VITE_AUDIT_API_PORT` | Audit Service 포트 번호 | 8083 | 아니오 |

## 참고

- 환경 변수는 `src/api/axiosInstance.ts`에서 사용됩니다.
- 기본값이 설정되어 있어 환경 변수를 설정하지 않아도 개발 서버가 실행됩니다.
