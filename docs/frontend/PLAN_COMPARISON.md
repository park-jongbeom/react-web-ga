  / → 랜딩 페이지 (간소화)

인증 후:

## 🎯 백엔드 협업 전략

### MVP 백엔드 병렬 개발
```
백엔드 Week 1 (10h) → Mock API 제공
                     ↓
프론트 Week 1 (24h) → Mock API로 UI 개발
                     ↓
백엔드 Week 2 (10h) → 실제 매칭 API 완성
                     ↓
프론트 Week 2 (24h) → 실제 API 연동
                     ↓
백엔드 Week 3~4 → Application API + 보안
프론트 Week 3 (12h) → 통합 테스트 + 최적화
```

### 전체 계획 백엔드 의존성
- Phase 2~3: User Profile API + AI 매칭 API
- Phase 4: 4개 프로그램 API + 학교 데이터
- Phase 5: 검색 API
- Phase 6: 문서 템플릿 API
- Phase 7: 미국 생활 정보 API (선택)

## 💡 선택 가이드

### MVP 계획을 선택해야 하는 경우 ⭐
- ✅ **빠른 시장 검증**이 필요한 경우
- ✅ **시간/예산 제약**이 있는 경우 (주당 10시간)
- ✅ **AI 매칭 기능**이 핵심 가치인 경우
- ✅ **백엔드와 병렬 개발**이 필요한 경우
- ✅ **기본 기능으로 사용자 피드백** 수집이 우선인 경우

### 전체 계획을 선택해야 하는 경우
- ✅ **모든 기능을 한번에** 구현해야 하는 경우
- ✅ **시간 여유**가 있는 경우 (16주 이상)
- ✅ **완성도 높은 플랫폼**이 필요한 경우
- ✅ **Phase 2 기능도 즉시** 필요한 경우
