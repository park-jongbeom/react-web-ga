# Problem 1-Pager: 매칭 결과 화면 구현

## 배경(Background)
- 프로토타입 데모에서 **프로필 입력 후 즉시 매칭 결과 화면**을 보여주는 흐름이 최우선 요구사항이다.
- 현재는 프로필 저장 후 대시보드로 이동하며, 매칭 결과 페이지가 별도로 존재하지 않는다.
- 백엔드 매칭 API(`POST /api/v1/matching/run`)는 완료된 상태로 실제 연동이 가능하다.

## 문제(Problem)
- 투자자 데모 플로우에서 핵심 가치(매칭 결과 시각화)가 즉시 노출되지 않아 설득력이 떨어진다.
- 매칭 API 응답 구조를 반영한 전용 UI/데이터 흐름이 구현되어 있지 않다.

## 목표(Goal)
- 프로필 입력 완료 후 **매칭 API 실행 → 매칭 결과 페이지 즉시 이동** 플로우 완성.
- 매칭 결과를 **Radar Chart + 추천 학교 카드**로 시각화.
- 백엔드 API 응답 구조(snake_case, ApiResponse 래퍼)와 완전 호환.
- TDD 방식으로 핵심 컴포넌트/서비스/페이지 테스트 포함.

## 비목표(Non-goals)
- 대시보드 전면 개편 및 타 페이지 UI 리뉴얼.
- 백엔드 API 스펙 변경 또는 서버 로직 수정.
- 북마크/지원현황 등 Week 4~5 예정 기능 구현.

## 제약(Constraints)
- Base UI 컴포넌트 및 디자인 토큰(`docs/ui-components.md`) 준수.
- 보안 체크리스트 반영: AuthN/AuthZ, CSRF, XSS, 에러 노출 차단 등.
- 기존 프로필 입력 UX는 유지하며 결과 이동만 추가.
- 에러/예외 처리 시 사용자에게는 일반화된 메시지만 노출.
